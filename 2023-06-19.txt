### 상품 페이지 만드는 단계

컨트롤러 - 이전에 작업한 폴더의 build.gradle 에서 thymeleaf, layout 복사해오기 - templates - product - register.html - 페이지 작동 확인(localhost:8080/product/register) - {파일 업로드/조회/삭제 확인 (+DB, 트랜잭션 등)} - axios lib : <script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script> 추가 - 옆에 upload 버튼 추가(upload 처리 후 초기화되어야함) - (ajax 이용시 FormData 사용) - 이벤트 걸기 uploadBtn, uploadInput에 클래스 ( 끝에 false, preventDefault, stopPropagation 적용 ) - 로그 찍어서 확인 (uploadInout) - 개발자 도구로 파일 여러개 업로드 후 확인 - Input 태그만 나옴 → console.dir로 변경 후 확인 - files ⇒ 이미지의 확장자를 체크하는 정규표현식 (gpt 이용) - formData 생성 (new 로 만들어야함) - for 루프 (버그 찾기 어려우므로 순서대로 하는게 좋음) - multiPartFormData로 보내야함 (ckck24 part.7 .. static-js-upload.js 6~13 / 공유드라이버← 택 , 2행 복사해서 추가) - axios.post(주소, formData, header) - 확인(html 에 파일 올리고 확인(네트워크 탭)) - localhost에 이미지 경로 올리고 확인

업로드 결과를 화면에 출력 - .then(res ⇒ {  }) 이용 - 콘솔.로그 확인 - div class(uploadDiv) 생성 - ul(uploadUL) 생성 - ul 변수화 - showProducts 함수 생성, 배열 ⇒ li -  for of 반복문 사용 ( https://jsdev.kr/t/for-in-vs-for-of/2938 참고 ) - 로그 확인 - 화면 확인 - let 으로 문자열 하나 생성 - 구조문의 할당  : const (fileName, link, uuid = uploadFile) - 문자열 += ` <li> 링크 </li> ` - uploadUL.innerHTML += str (← 기존에 파일에 게속 추가 할 수 있게끔) - <li> 안에 <div> 만들어서 링크 감쌈 - <p>파일이름, X버튼</p> 생성 - <style>.uploadUL {list, display}, .uploadUL li {margin, background-color, padding, border, border-radios}</style>

새 탭으로 원본파일 만드는 링크 - <a href = ` 경로 ~ ${uuid}, ${fileName}` target =”_black”> <img ~~~> </a>
